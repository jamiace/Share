Hahow 線上學習平台 AI 輔助系統

輕量化產品企劃書｜提升完課率與學習成效的解決方案

專案核心理念：
從「考高分」的高中生市場轉向「求即戰力、克服拖延」的成人學習市場。產品定位為「影音教材的防腐劑與調味專家」，以最低的 Token 成本與最輕量的認知負載，將學生的注意力精準導回影片本身，達成「完課率」與「確保學會」的雙重目標。

一、 產品設計原則與技術架構

為了解決 AI 資源濫用導致成本失控，以及避免本末倒置干擾影音學習體驗，本專案導入嚴格的限制與快取機制。透過完善的前後端 API 協作與高效能的題庫資料庫部署，確保服務穩定且輕量：

  • 一次性批次生成（Batch Processing）： LLM 僅在課程上架時讀取教材文本，預先生成所有「隨堂點心題」與「章節摘要」並寫入資料庫緩存，將呼叫次數從「用戶觀看數」降至「課程數」。
  • 嚴格使用量控制（Quota 鎖）： 捨棄無限制發問，設定每位用戶單日最高 3 次對話額度，引導用戶善用既有影片資源。
  • 完全依附影音體驗： AI 工具作為輔助組件內嵌，不設立搶奪流量的獨立入口，一切互動皆指向「引導觀看影片進度」。

二、 產品功能規劃 (MVP)

1. Web 端內嵌式「精簡版隨堂組件」

功能描述： 內建於 Hahow 影片播放器旁的輕量化外掛模組。

  • 觀念隨堂點心： 影片播放至特定節點（如 50%），跳出預先生成的單選題。答錯不作長篇解釋，而是引導：「建議回到影片 03:15 處重新觀看。」
  • AI 影音書籤： 將預生成的 300 字章節精華置於網頁側邊欄，方便快速抓取重點。

2. LINE「完課衝刺」輕量通知 Bot

功能描述： 專注於進度追蹤與痛點誘因推播，不作耗能的深度對練。

  • 防囤課機制： 偵測停滯觀看達 3 天，即發送 50 字內預先生成的「微劇透」或「實用痛點金句」。
  • 一鍵喚醒： 推播內容附帶直達 Hahow 該章節播放頁面的連結，以最低阻力促發行動。

三、 目標客群 (Target Audience)

初期專注鎖定 Hahow 上具備明確痛點的白領在職進修族群，避開過度發散：

硬實力技術剛需者
(程式/數據類)		 實作卡關易放棄，需要精準驗證觀念。 隨堂點心直接點出影片回顧秒數，快速解惑不迷失。

四、 商業模式推動策略

採階段性推進，從單點驗證走向全域授權：

  1. B2B2C 課程加價購（概念驗證期）： 與特定熱門課程（如 Python、行銷管理）老師合作，購課時提供「加購 AI 專屬導師/題庫服務」，由開發團隊、老師、平台三方分潤。
  2. B2B 企業端加值服務（獲利成長期）： 包裝為「企業人資考核與完課保證模組」導入 Hahow for Business，為企業端提供去識別化的學習成效微型看板，收取高溢價。
  3. B2B SaaS 平台級授權（規模化期）： 以「訂閱年費 + MAU」形式，將底層題庫與批次生成架構授權給 Hahow 作為全站 VIP 標配功能。

五、 專案時程與成本預估 (6-8 週敏捷開發)

基於輕量化與高緩存架構，大幅壓縮研發與營運成本：

  • W1-W2 核心架構建置： 教材 Ingestion Pipeline、撰寫 Prompt 批次生成腳本、資料庫快取建置。
  • W3-W4 前端與 API 串接： Web 內嵌 UI 開發、Quota 鎖機制建立、LINE Bot 基礎推送 API 整合。
  • W5-W6 內部整合測試： 擇定 3~5 堂指標課程進行跑模，驗證題目精準度與答錯引導秒數。
  • W7-W8 MVP 上線： 數據看板建置，正式對小規模使用者進行封測與投放。

* 營運 Token 成本因採「批次預處理」模式，僅需支付極低的單次生成費，不再隨使用者觀看次數呈線性暴增。
